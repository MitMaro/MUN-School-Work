Tim's PyCal Documentation
Introduction

This calculator supports two different forms of mathematical calculations. Calculations using operators and calculations using functions. For every calculation done, stored in history are the operation and result that can be recalled later. A two line display shows the current result on the first line and the operation being performed on the second. Input to the calculator is done using the buttons on the interface. For some input a keyboard can also be used for input.

Numerical Input

The calculator supports full numerical input with scientific notation. The calculator doesn't allow invalid input (you can't have two decimals for example). Negation of a number is done by the ± button. Pressing the equals button assigns the current numerical input to the result.

Scientific Notation

Scientific notation is inputed using the `EE` button negative scientific notation is also supported.
Example: Pressing the 1, 2, 3, EE and then 2 buttons will show 1.23E2 which is the same as 123x102

Two Line Display

The calculator has a two line display, the first line show the current result and the second line show the current user input. If the text in the display become to large to be shown then '>>' and/or '<<' will be shown to the left and/or right of the text. You can scroll the display using the left and right arrow keys on your keyboard.

Operational Math

The calculator supports the addition (+), subtraction (-), multiplication (*), division (/), modulo (%) and power (^) operations. The operation is always performed on the current result. For example if the current result is 5 and you press the + button and the input 3 the result will be 8.

Functional Math

Support for mathematical functions is provided within the calculator. A list of the mathematical functions provided within the calculator is provided below. When a function only requires one input (argument) then you press the function button, input the number and then the equals button to get a result. For example the Sin function takes one argument as input. To calculate the sine of 0.5 you press the Sin function button then enter 0.5 and press enter.

When a function requires more then one argument you press the argument separator button (,) to enter the next argument (repeat the process for any additional arguments that need to be inputed). For example the Log function requires two arguments, the base and a number, to calculate the base 10 log of 3 you press the Log button then input 10, you then press the argument separator button and input 3 and hit enter to get the result.

The calculator will not allow you to input more arguments then is required for any particular function. An ARGUMENTS ERROR message will be displayed if to few arguments are supplied.

Clear, Clear Entry, Delete and Exit

The calculators clear button (C) will remove all user input and reset the result to 0. It resets the calculator back to its state when it was first started. However, history is not cleared.

The clear entry button (CE) will clear the current user input and errors. Nothing else is cleared.

The delete button (Del) will delete one character of user input at a time. This allows you to go back and fix an error that you may have made. When you are using history for input, delete will clear the whole entry.

To exit the calculator just press the Off button.

Result and Operational History

The calculator supports full result and operational history, what this means is any operation you perform the result of the operation as well as that operation is stored in history and can be recalled. To access the histories your use the next (<<) and previous (>>) history buttons for each history. From this point onwards everything discussed can be applied to the operational and result histories. Pressing the previous button will display the last history and pressing it again will display the history before that and so on. The next button is used for going forwards in history. The calculator remembers your current input and pressing the next history button enough will bring back your current input. There are several limitations on using history however. Pressing any clear button while viewing history will clear both your history and current input and you can not modify the input while viewing history.
