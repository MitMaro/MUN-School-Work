Tim's PyCal Source Documentation
Introduction

I tried to implement all the bonus features I could and I believe the only additional features I did not implement are the undo last operations and the internally stored quantities. These features however are not needed in the way the calculator is designed. Although you can not undo the last operation you can retrieve the previous result using the result history. The result history can also be used to store values by simply inputting a value and hitting enter.

Another thing to note is that when looking at the source code things may seem more complicated then they need be. This is generally true, as the code is written to be dynamic. I started writing this code with the purpose of adding a feature where the user of the calculator, if they had knowledge of Python, could write their own mathematical functions and add them to the calculator. I didn't have time to add this feature but it wouldn't take long to add it.

The Source Files

In reading the source for this application I would suggest reading them in the order provided below to ensure your not reading code that is referencing code yet unread.

CalcWidgets.py

I wrote all the buttons in the calculator in groups. This file contains all the GUI megawidgets that are used in the calculator. Mostly they are just groups of buttons that when clicked generate a virtual event that can be later captured in the main application. The only exception to this is the display which is only used for display purposes only. All key bindings are also done within this file. I decided to generate virtual events in place of using callback functions because I found the callbacks were added a lot of bulk to the application.

CalcFunctions.py

A very small file containing several mathematical functions that are used by the calculator to calculate a result.

History.py

This file contains a class that handles the storing and iteration of histories. The standard iterator in Python doesn't allow backwards iteration and I needed a way to have the iterative index to start at -1 for the purpose of displaying the current value. The History class also stores the current input when the history is being iterated.

Number.py

If I had my time back I would have called this class NumericalInput. The Number class allows for the creation of a valid number one character at a time. The class stops there being two decimals in a given input, no decimal in the exponent part and handles unary minus. This class will always produce a string that is valid for parsing by both Decimal and Float.

CalculatorValues.py

Certain functions (the Log function for example) take multiple user inputs. The class contained within the CalculatorValues.py file allows for easy creation, manipulation and accessing of these values. It assures the data is always in the correct type, makes sure there is always at least one value in its internal list and provides iteration over the list of values.

Calculator.py

This file is the heart of the calculator. It contains all the necessary functions to handle user input, storing and retrieving histories, function argument handling, calculating both functional operational operations and formatting and displaying of the input and result.

addmachGUI.py

The various calculator classes are useless if the user doesn't have a way to interact with them. The addmachGUI.py file creates a user interface and links the virtual events from the megawidgets to their corresponding methods in the calculator class.
